# GoReleaser Integration Implementation Plan

## Overview

Add GoReleaser to freedius to automate building and publishing static binaries for 6 platforms (linux/darwin/windows Ă— amd64/arm64) on `git tag`. This replaces the current manual `go build` / `docker build` workflow and enables `go install @latest` to display the correct version. Docker image building remains separate (via the existing Dockerfile + mage targets).

## Current State Analysis

- `cmd/freedius/main.go:45` â€” `var version = "dev"` is hardcoded; `--version` always prints "dev"
- `cmd/freedius/main.go:331-343` â€” `handleEarlyArgs` prints `version` directly for `--version`
- `magefiles/mage.go:139-142` â€” `Build` target is bare `go build -o freedius ./cmd/freedius` with no flags, no version injection
- `magefiles/mage.go:520-551` â€” Docker targets exist but are local-only, no published image

- `tools.go:1-7` â€” only imports `magefile/mage`, no release tooling
- `go.mod:1` â€” module path `github.com/pfrack/freedius`, Go 1.26.5
- No `.goreleaser.yaml` exists anywhere in the repo

## Desired End State

When a maintainer pushes a git tag `v0.1.0`:
1. GitHub Actions release workflow triggers
2. GoReleaser builds 6 static binaries with correct version injection
3. Archives (tar.gz for Unix, zip for Windows) + checksums are uploaded to GitHub Releases
4. `freedius --version` prints the tag (e.g., `v0.1.0`) for both GoReleaser builds and `go install @latest`
5. `mage build` still works for local development

### Key Discoveries:

- The `debug.ReadBuildInfo()` approach (research.md:283-299) lets `go install @latest` show the module version without ldflags â€” GoReleaser still injects via ldflags, but the fallback covers non-GoReleaser builds
- The ldflags path must be fully qualified: `github.com/pfrack/freedius/cmd/freedius.version` (research.md:185, 212)
- Build flags should match the Dockerfile exactly: `CGO_ENABLED=0`, `netgo,osusergo`, `-s -w`, `-trimpath` (Dockerfile:8-13)
- GoReleaser's `{{.Version}}` comes from the git tag (e.g., `v0.1.0`) â€” the tag must include the `v` prefix

## What We're NOT Doing

- Homebrew tap (requires separate tap repo, auto-generated by GoReleaser but out of scope)
- Scoop bucket (Windows package manager, out of scope)
- Docker image publishing via GoReleaser (keeping Docker separate per user decision)
- Self-update subcommand (`freedius update`) â€” out of scope
- `install.sh` script â€” out of scope

## Implementation Approach

1. First, fix version detection in `main.go` so `--version` works correctly regardless of build method
2. Create `.goreleaser.yaml` with build config matching the Dockerfile's established flags
3. Create `.github/workflows/release.yml` triggered on git tags
4. Update magefiles to support local goreleaser testing

## Critical Implementation Details

- **Version variable path**: The ldflags `-X` flag must reference `github.com/pfrack/freedius/cmd/freedius.version` â€” this is the fully qualified path to the package-level `version` variable in `cmd/freedius/main.go:45`. GoReleaser's `{{.Version}}` resolves to the git tag (e.g., `v0.1.0`).
- **debug.ReadBuildInfo() fallback**: When `version` is still `"dev"` (non-GoReleaser build), fall back to `debug.ReadBuildInfo().Main.Version`. For `go install @latest`, this returns the module version (e.g., `v0.1.0`). For `go run .` or `go build`, it returns `(devel)`.
- **Build flag parity**: GoReleaser's `CGO_ENABLED=0`, `netgo,osusergo` tags, `-s -w` ldflags, and `-trimpath` must match the Dockerfile exactly to produce identical binaries.
- **Tag format**: GoReleaser expects `v`-prefixed semver tags (e.g., `v0.1.0`). The release workflow triggers on `push: tags: ['v*']`.

## Phase 1: Version Detection in main.go

### Overview

Add a `getVersion()` function that uses `debug.ReadBuildInfo()` as a fallback when the `version` variable is still `"dev"`. Update the `--version` handler to use it. This ensures `go install @latest` displays the correct version without requiring ldflags injection.

### Changes Required:

#### 1. Add `getVersion()` function

**File**: `cmd/freedius/main.go`

**Intent**: Create a function that returns the injected version if set (by GoReleaser ldflags), otherwise falls back to `debug.ReadBuildInfo()` for `go install @latest` support.

**Contract**: Function signature `func getVersion() string`. If `version != "dev"`, return `version`. Otherwise, try `debug.ReadBuildInfo()` and return `info.Main.Version` if it's not `(devel)` or empty. Return `"dev"` as final fallback.

```go
func getVersion() string {
	if version != "dev" {
		return version
	}
	if info, ok := debug.ReadBuildInfo(); ok {
		if v := info.Main.Version; v != "(devel)" && v != "" {
			return v
		}
	}
	return "dev"
}
```

#### 2. Update `--version` handler

**File**: `cmd/freedius/main.go:331-343`

**Intent**: Replace the direct `version` reference in `handleEarlyArgs` with `getVersion()`.

**Contract**: Line 334 changes from `fmt.Printf("freedius %s\n", version)` to `fmt.Printf("freedius %s\n", getVersion())`.

#### 3. Add `runtime/debug` import

**File**: `cmd/freedius/main.go:5-27`

**Intent**: Add the `runtime/debug` import needed for `debug.ReadBuildInfo()`.

**Contract**: Add `"runtime/debug"` to the import block.

## Phase 2: Create .goreleaser.yaml

### Overview

Create the GoReleaser configuration that builds 6 static binaries (linux/darwin/windows Ă— amd64/arm64) with flags matching the Dockerfile, produces tar.gz/zip archives, generates checksums, and configures GitHub Releases.

### Changes Required:

#### 1. Create .goreleaser.yaml

**File**: `.goreleaser.yaml` (new file)

**Intent**: Configure GoReleaser to build, archive, and release freedius binaries on git tag.

**Contract**: The YAML must include:
- `builds` section with `CGO_ENABLED=0`, `netgo,osusergo` tags, `-s -w` ldflags with version/commit/date injection, `-trimpath`, `goos: [linux, darwin, windows]`, `goarch: [amd64, arm64]`
- `archives` section with tar.gz for Unix, zip for Windows
- `checksum` section
- `release` section with GitHub owner/repo
- `before.hooks` with `go mod tidy`
- `env` section (optional, for Go version)

```yaml
version: 2

before:
  hooks:
    - go mod tidy

builds:
  - env:
      - CGO_ENABLED=0
    flags:
      - -tags=netgo,osusergo
      - -trimpath
    ldflags:
      - -s -w -X github.com/pfrack/freedius/cmd/freedius.version={{.Version}}
      - -X github.com/pfrack/freedius/cmd/freedius.commit={{.Commit}}
      - -X github.com/pfrack/freedius/cmd/freedius.date={{.Date}}
    goos:
      - linux
      - darwin
      - windows
    goarch:
      - amd64
      - arm64

archives:
  - format: tar.gz
    name_template: "{{ .ProjectName }}_{{ .Version }}_{{ .Os }}_{{ .Arch }}"
  - format: zip
    name_template: "{{ .ProjectName }}_{{ .Version }}_{{ .Os }}_{{ .Arch }}"
    goos:
      - windows

checksum:
  name_template: "{{ .ProjectName }}_{{ .Version }}_checksums.txt"

release:
  github:
    owner: pfrack
    repo: freedius
```

### Success Criteria:

#### Automated Verification:

- `goreleaser check` passes (validates config syntax)
- `goreleaser build` (dry-run, local) produces 6 binaries
- `go mod tidy` succeeds (before hook)

#### Manual Verification:

- `goreleaser check` output shows no errors
- Local `goreleaser build --clean` produces binaries in `dist/` with correct names

## Phase 3: Create Release CI Workflow

### Overview

Create a GitHub Actions workflow that triggers on `git tag` push (matching `v*`), installs GoReleaser, and runs `goreleaser release` to build binaries and publish to GitHub Releases.

### Changes Required:

#### 1. Create .github/workflows/release.yml

**File**: `.github/workflows/release.yml` (new file)

**Intent**: Define the CI workflow that runs GoReleaser on tag push.

**Contract**: The workflow must:
- Trigger on `push: tags: ['v*']`
- Use `actions/checkout@v4` with `fetch-depth: 0` (needed for GoReleaser to determine version)
- Use `actions/setup-go@v5` with Go 1.26.4 (matching ci.yml:24)
- Use `goreleaser/goreleaser-action@v6` with `version: latest` and `args: release`
- Set `GITHUB_TOKEN` for GitHub Releases permissions

```yaml
name: release

on:
  push:
    tags:
      - 'v*'

permissions:
  contents: write

jobs:
  release:
    runs-on: ubuntu-latest
    timeout-minutes: 15
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0
      - uses: actions/setup-go@v5
        with:
          go-version: '1.26.4'
          cache: true
      - name: Run GoReleaser
        uses: goreleaser/goreleaser-action@v6
        with:
          version: latest
          args: release
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
```

### Success Criteria:

#### Automated Verification:

- Workflow YAML is valid (can be checked with `actionlint` if available)
- `actions/checkout@v4` and `actions/setup-go@v5` versions match ci.yml
- Go version matches ci.yml:24 (`'1.26.4'`)
- `fetch-depth: 0` is set (GoReleaser needs full git history)

#### Manual Verification:

- Workflow appears in GitHub Actions tab when a `v*` tag is pushed
- On tag push, GoReleaser builds all 6 platforms and creates a GitHub Release
- Release page shows archives + checksums for all platforms

#### 2. Add Goreleaser mage target

**File**: `magefiles/mage.go` (after DockerPush, before Help2)

**Intent**: Add a `Goreleaser` target that runs `goreleaser` locally, installing it if missing (following the pattern of `LintStatic`, `LintGolangci`, `Govulncheck`).

**Contract**: Function `func Goreleaser() error` that:
- Checks if `goreleaser` is installed via `which goreleaser`
- If missing, installs via `go install github.com/goreleaser/goreleaser/v2@v2.6.2`
- Runs `goreleaser` with any extra args from `ARGS` env var
- Returns error on failure

```go
func Goreleaser() error {
	if _, err := sh.Output("which", "goreleaser"); err != nil {
		if err := sh.RunV("go", "install", "github.com/goreleaser/goreleaser/v2@"+toolVersionGoreleaser); err != nil {
			return err
		}
	}
	args := []string{}
	if extra := os.Getenv("ARGS"); extra != "" {
		args = append(args, strings.Fields(extra)...)
	}
	return sh.RunV("goreleaser", args...)
}
```

#### 3. Update Build target to support version injection (optional)

**File**: `magefiles/mage.go:139-142`

**Intent**: Enhance the `Build` target to optionally inject version from `git describe --tags`, matching GoReleaser's ldflags behavior for local builds.

**Contract**: Update `Build()` to use `git describe --tags --always --dirty` to get the version, then pass it via `-ldflags`. If git describe fails (no tags), fall back to the existing bare build.

```go
func Build() error {
	fmt.Println("â†’ Building freedius binary...")
	version, _ := sh.Output("git", "describe", "--tags", "--always", "--dirty")
	version = strings.TrimSpace(version)
	if version == "" {
		version = "dev"
	}
	return sh.RunV("go", "build",
		"-o", "freedius",
		"-ldflags", fmt.Sprintf("-s -w -X github.com/pfrack/freedius/cmd/freedius.version=%s", version),
		"-tags", "netgo,osusergo",
		"-trimpath",
		"-gcflags=-l",
		"./cmd/freedius",
	)
}
```

- Each binary's `--version` shows the correct version

### Manual Testing Steps:

1. Run `mage build` and verify `./freedius --version` shows a version (git tag or "dev")
2. Run `goreleaser check` and verify no config errors
3. Run `goreleaser build --clean` and verify 6 binaries + archives in `dist/`

- GoReleaser builds are parallelized across platforms; 6 builds should complete in under 5 minutes on GitHub Actions runners
- `CGO_ENABLED=0` produces truly static binaries matching the project's "zero external dependencies" promise
- `-trimpath` ensures reproducible builds by removing local filesystem paths

- The magefiles `Build` target enhancement is backward-compatible (falls back to bare build if git describe fails)
